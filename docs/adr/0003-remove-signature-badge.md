# 0003. SIGNATURE 배지를 제거하고 카테고리로 통일한다

**Status:** Proposed  
**Date:** 2026-05-07

## Context

메뉴 항목에는 `is_signature` 불리언 컬럼(구)과 `tag` 컬럼의 `SIGNATURE` 값(신)이 있다.
동시에 `category = '시그니쳐'`라는 카테고리 그룹도 존재한다.
두 가지가 같은 개념을 표현하므로 관리자가 카테고리와 배지를 이중으로 관리해야 한다.

## Decision

`SIGNATURE` 배지(tag 값 및 `is_signature` 컬럼)를 제거한다.
대표 메뉴 구분은 `category = '시그니쳐'` 하나로 통일한다.
배지는 시간 기반 정보(`NEW`, `SEASON`)만 유지한다.

## Trade-offs

**이점:** 중복 개념 제거. 관리자가 카테고리 하나만 관리하면 됨.  
**포기하는 것:** "커피 카테고리 안의 시그니쳐 메뉴"처럼 카테고리를 가로지르는 강조 표시 불가.
현재 그런 요구가 없으므로 YAGNI.
