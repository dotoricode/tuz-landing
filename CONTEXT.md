# Tuz 카페 도메인 용어집

울산 중구에 위치한 Tuz(투즈) 카페의 랜딩 페이지 도메인 언어.

---

## 핵심 용어

### Owner (사장)

카페의 공동대표. 항상 두 명이며 각자 고유한 역할 문맥을 가짐.

- **큰 사장** — 공동대표 1. 코드에서 `barista = '큰 사장'`으로 식별.
- **작은 사장** — 공동대표 2. 코드에서 `barista = '작은 사장'`으로 식별.

두 Owner는 각자 독립적으로 오늘의 Pick을 선정한다.

### Pinned Notice (홈 화면 고정 공지)

News 항목 중 하나를 홈 화면 상단 마퀴에 고정하는 기능.
DB 컬럼명: `is_pinned` (구 `is_today` — 자동 만료 없이 수동 해제 전까지 유지되므로 변경).
동시에 하나만 고정 가능. 저장 시 다른 항목의 `is_pinned`가 자동 해제됨.

### Trend Report (트렌드 리포트)

관리자 전용(`/trend/`, noindex) 시장 조사 도구. 외부 스크립트가 주기적으로 생성해 Supabase Storage `reports` 버킷에 업로드한다.
Owner들이 메뉴 기획 결정에 활용한다.

데이터 소스: 외부 커뮤니티, SNS 탐색 (내부 POS·판매 데이터 없음).

리포트 카테고리:
- **현재 인기 트렌드** — 지금 뜨고 있는 음료·메뉴
- **앞으로 인기 있을 트렌드** — 선제적으로 캐치한 예측 트렌드
- **식어가는 트렌드** — 피크를 지난 메뉴

### Stamp Event (쿠폰 추첨 이벤트)

쿠폰을 다 채운 방문객 중 랜덤 추첨으로 5명을 선정하는 이벤트.
당첨자는 `winners` 테이블에 닉네임·혜택·기간으로 저장되고, EVENT 타일을 통해 공개된다.
**현재 기획 중 / 미정** — EVENT 타일은 주석 처리 상태이며 활성화 시점 미정.
당첨자 수는 항상 5명으로 고정.

### Menu

카테고리(`category`)로 그룹핑되는 메뉴 항목 목록. 현재 카테고리: 시그니쳐 / 커피 / 논커피 (변동 가능).
각 항목에 시간 기반 배지(`NEW`, `SEASON`)를 붙일 수 있다.
`SIGNATURE` 배지는 "시그니쳐" 카테고리와 중복이므로 제거 (ADR-0003).
`is_signature` 컬럼도 함께 제거 예정.

### Greeting (인사말)

카페 공동 인사말. Owner 개인이 아닌 Tuz 카페 이름으로 하나만 존재 (DB 단일 행, `id = 1`).
기본 서명: `— TUZ 드림`.

### Schedule Notice (영업시간 공지)

News 태그 `SCHEDULE` (구 `HOURS`). 영업시간 변경을 방문객에게 안내하는 공지문.
실제 영업시간 계산은 `settings` 테이블이 담당하며, Schedule Notice는 그와 독립적인 안내 텍스트다.
`settings` 저장 시 변경이 감지되면 Schedule Notice 생성 모달이 자동으로 뜬다 (ADR-0002).

### Pick (오늘의 추천)

각 Owner가 수동으로 선정하는 오늘의 메뉴 추천. 항상 Menu에 실제로 존재하는 항목이어야 한다.
Owner별로 한 개씩, 동시에 두 개가 표시된다.

- Pick은 Menu를 ID로 참조한다 (ADR-0001).
- `note` (한줄 코멘트)는 Pick 고유 정보로, Menu에서 상속되지 않는다.
- `photo`는 선택적 오버라이드. 비어 있으면 Menu의 `hero_photo`로 폴백한다.
